# PHP反序列化一键利用工具

一个用于自动分析PHP反序列化漏洞、发现POP链并自动利用的Python工具。

## 功能特性

- **一键利用**: 只需提供目标URL，自动完成分析、生成、利用全流程
- **自动参数检测**: 自动尝试常见参数名 (ser, data, payload, x等)
- **自动绕过__wakeup**: 检测到__wakeup时自动修改属性数量绕过
- **自动FLAG提取**: 自动识别并提取FLAG
- **交互式输入**: 运行时提示输入文件路径或命令

## 安装要求

```bash
Python 3.7+
```

无需额外依赖，使用Python标准库。

## 使用方法

### 一键利用（推荐）

```bash
# 最简单用法 - 提供目标URL
python php_unserialize_tool.py http://target.com/

# 本地源码 + 远程利用
python php_unserialize_tool.py -f vuln.php http://target.com/
```

### 运行时交互

```
[!] 漏洞类型: FILE_READ (文件读取)
[?] 输入要读取的文件路径 (直接回车尝试常见位置): 
```

- **直接回车**: 自动尝试常见位置 (`/flag`, `flag.php`, `fllllllag.php`等)
- **输入路径**: 只尝试指定路径

### 命令行参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `target` | 目标URL (必填) | `http://target.com/` |
| `-f, --file` | 本地PHP源码文件 | `-f vuln.php` |
| `--cmd` | RCE命令 | `--cmd "cat /flag"` |
| `--read` | 要读取的文件 | `--read /etc/passwd` |
| `--param` | 指定参数名 | `--param x` |
| `--method` | 请求方法 (默认REQUEST) | `--method POST` |

### 高级用法

```bash
# 手动指定参数名
python php_unserialize_tool.py -f vuln.php http://target.com/ --param x

# 手动指定文件路径（跳过交互）
python php_unserialize_tool.py -f vuln.php http://target.com/ --read fllllllag.php

# 手动指定RCE命令
python php_unserialize_tool.py -f vuln.php http://target.com/ --cmd "cat /flag"
```

### 手动模式

```bash
# 手动生成payload（不发送）
python php_unserialize_tool.py -c "User" -p "name:admin,cmd:id"
```

## 自动功能

| 功能 | 说明 |
|------|------|
| 参数检测 | 自动尝试 `ser`, `data`, `payload`, `x`, `str`, `input`, `var`, `p` |
| 方法尝试 | 自动尝试 GET 和 POST 请求 |
| __wakeup绕过 | 检测到__wakeup时自动修改属性数量为999 |
| FLAG提取 | 自动匹配 `NSSCTF{...}`, `flag{...}`, `ctf{...}` 格式 |
| 常见文件 | `/flag`, `/flag.txt`, `flag.php`, `fllllllag.php`, `/etc/passwd` |
| 常见命令 | `id`, `ls`, `cat /flag`, `ls /` |

## 支持的漏洞类型

### RCE (远程命令执行)
- `system()`, `exec()`, `shell_exec()`, `passthru()`
- `eval()`, `assert()`, `popen()`, `proc_open()`

### FILE_READ (文件读取)
- `file()`, `file_get_contents()`, `highlight_file()`
- `show_source()`, `readfile()`, `include()`

### FILE_WRITE (文件写入)
- `file_put_contents()`, `fwrite()`

## 支持的魔术方法

### 入口点
- `__destruct()` - 对象销毁时触发
- `__wakeup()` - 反序列化时触发
- `__toString()` - 对象转字符串时触发

### 链接方法
- `__call()` - 调用不存在方法时触发
- `__get()` - 访问不存在属性时触发
- `__set()` - 设置不存在属性时触发
- `__invoke()` - 对象作为函数调用时触发

## POP链追踪

工具支持自动追踪多跳POP链:

```
TianXiWei::__wakeup()
    ↓ $this->ext->nisa($this->x)
Ilovetxw::__call()
    ↓ $this->huang->fun = $arg[0]
four::__set()
    ↓ strtolower($this->a)
Ilovetxw::__toString()
    ↓ $bb = $this->su; $bb()
NISA::__invoke()
    ↓ eval($this->txw4ever)
RCE!
```

## 属性可见性处理

| 可见性 | 序列化格式 | 示例 |
|--------|------------|------|
| public | `s:N:"name"` | `s:3:"cmd"` |
| protected | `s:N:"\0*\0name"` | `s:7:"\0*\0cmd"` |
| private | `s:N:"\0Class\0name"` | `s:8:"\0User\0cmd"` |

## 注意事项

1. **目标URL**: 必须提供可访问的目标URL
2. **源码获取**: 如果目标不返回源码，需使用 `-f` 指定本地文件
3. **参数名**: 工具会自动尝试常见参数名，也可手动指定
4. **FLAG格式**: 支持 `NSSCTF{...}`, `flag{...}`, `ctf{...}` 等格式

## 局限性

- 不支持动态属性名分析
- 不支持回调函数参数追踪
- 不支持闭包/匿名函数分析
- 需要完整的PHP类定义
- 从URL加载源码时，目标需返回PHP源码而非执行结果
